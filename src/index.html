Git/Github

Git의 필요성

Git이란 파일 버전을 관리하는 분산 형상 관리 시스템입니다. 직접 파일을 주고 받는 방식보다 효율적이며, 유의미한 변경 사항을 기록하고 병합할 수 있습니다. 보다 문제 상황에 유연하게 대응할 수 있도록 이용자 간의 규칙을 정하는 것이 일반적입니다.

기본 개념과 용어

  • repository: 저장소. remote/local
  • working directory: 작업 공간(untracked)
  • staging area: 추적중인 상태
  • branch: 분기점. 가상의 나뭇가지
  • HEAD: 브랜치의 마지막 커밋을 바라보는 포인터
  • ignore: 추적 관리하지 않을 파일을 정의
  • conflict: 같은 라인의 변경이 겹처 중첩된 경우 충돌

명령어

  • git init:저장소 초기화. 추적 시작
  • git clone 저장소주소: 기존에 존재하는 저장소 복제
  • git log:변동 내역 조회
  • git status:파일들의 상태조회
  • git diff: 특정 기준을 기준으로 내용 비교
  • git add: 대상의 변경 사항을 추적관리 시작(스테이징)
  • git commit: 파일의 유의미한 변경점을 기준으로 스냅샷
  • git fetch: 변동 내역을 가져오나 로컬에 반영되지 않음. 병합 전 확인 용도로 사용
  • git pull: 변동 내역 받아오기
  • git push: 변동 내역을 원격 저장소로 보내기
  • git branch: 저장소에 존재하는 분기 목록 조회. 또한 생성/병합/삭제/변경 가능
  • git checkout 저장소명: 브랜치로 이동. switch
  • git remote: 원격 저장소 조회. 단축 이름으로 추가 가능
  • git reset: 변경 사항 되돌리기. 옵션에 따라 실행 범위 다름
  • git revert: 변동 사항을 되돌리나 버전은 유지(commit 남음)
  • git rebase: 내가 작업중인 브랜치를 main 마지막 commit을 기준으로 한다는 개념
